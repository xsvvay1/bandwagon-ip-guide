## 怎么判断IP是不是真的被封了

很多人连不上就以为IP被封，实际上可能只是网络波动或者服务器临时维护。

在KiwiVM后台有个官方检测工具，把VPS-ID替换进去就能直接查：


https://kiwivm.64clouds.com/VPS-ID/main-exec.php?mode=blacklistcheck


VPS-ID在KiwiVM主控台页面上能找到，是一串数字。

另外也可以用ping工具从多个地点测试你的IP是否有响应。如果多个地点都显示不通，基本就是封了；如果国外节点通、国内不通，大概率是GFW封了这个IP。

确认IP被封了，再来考虑怎么处理。

换完IP之后如果还是连不上，重启一次VPS，或者重新检测下IP状态——有时候是本地DNS缓存没刷新。

### 方法二：切换机房（免费，但有条件）

每个搬瓦工套餐默认支持多个机房，在KiwiVM面板里可以免费一键迁移到其他机房，迁过去之后IP就变了。

**前提条件**：你的IP必须还没被封。IP已经被封的话，机房迁移功能会提示不可用，这条路就走不通了。

免费切换机房的几个注意点：

- 数据完整保留，不需要重新装系统
- 迁移本身免费，但会消耗一定流量
- 从A机房切到B机房再切回A机房，回去之后IP通常还是原来那个——所以别指望"切一圈回来就换了IP"
- 不同套餐可选的机房数量不一样，CN2 GIA-E套餐可以选十几个机房，基础KVM套餐选项少一些

说实话，这个方法主要适合**预防性换IP**——IP还能用，但想试试其他机房的线路质量，或者主动切一个更稳的机房。等IP真封了，只剩付费这条路。

## 付费换IP之后：新IP收到后该做什么

换完IP之后几个步骤别漏掉：

1. **更新SSH连接配置**：本地的SSH config文件、密钥对、登录记录里都有旧IP，手动改一下
2. **更新DNS解析**：如果VPS绑了域名，去域名服务商那边把A记录换成新IP，等待解析生效（通常几分钟到几小时）
3. **更新防火墙白名单**：如果你用IP白名单控制SSH访问权限，记得把新IP加进去
4. **重启确认**：换完IP后重启VPS，确认新IP正常响应
